A Hybrid Model for Security & Stability

Vihaan's architecture separates network security from currency issuance, using a high-performance DAG ledger and Randomized Proof of Stake.

Beyond Blockchain: The Block-Lattice

Individual Financial Ledgers

Instead of a single, congested blockchain, every user has their own personal blockchain called an "account-chain". Only the owner can add new transactions to their chain.

ü§ù

Asynchronous Transactions

A transaction is a simple handshake between two account-chains: a `send` block on the sender's chain and a `receive` block on the recipient's. This allows for massive parallel processing.

This architecture eliminates the need for a mempool or competitive block creation, resulting in a near-instantaneous, lightweight, and highly scalable network.

The Two-Tiered Node System

Guardians (Validator Nodes)

These are dedicated nodes run by staked community members that form the network's security backbone. They listen for, validate, and confirm all transactions on the network.

üì±

Sovereign Wallets (Light Nodes)

Your wallet is your personal, lightweight node. It signs all transactions locally on your device, giving you ultimate control. You never need to trust a third-party server to send funds.

Consensus: Randomized Proof of Stake

In the rare event of a conflict, a randomly selected committee of staked Guardians votes to secure the network. This model is efficient, secure, and decentralized.

üí∞

Staking as a Security Deposit

To become a Guardian, users must lock VHC as a security deposit. This stake can be slashed (destroyed) for malicious behavior, ensuring validators have significant financial "skin in the game."

üé≤

The Random Guardian Committee

Each day, a new committee of 100 Guardians is randomly selected from the entire pool of stakers. This constant rotation makes it extremely difficult for an attacker to predict, bribe, or attack the active validators.
